# Systems Workspace

This folder contains reusable models for ERP architecture, manufacturing operations, reporting, planning, and leadership reasoning.

## Core Ideas

- Operational determinism
- ERP as digital constitution
- Single source of truth
- Master data as strategic asset
- Physical reality before system abstraction
- Exception visibility
- Governance as architecture

## Purpose

The systems workspace turns repeated experience into reusable operating models.

A good systems artifact should clarify:

- what is happening in reality
- what the system says is happening
- where the mismatch appears
- what decision is needed
- how the improvement becomes durable
